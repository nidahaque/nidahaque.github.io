Day 1

Foundations & Prompting

Context LLMs Prompting

Topics covered

  • LLM context layers: training → alignment → system prompt → user
    Think of an LLM like an iceberg. At the base is training data — the vast internet text the model learned from, giving it general world knowledge. Above that is alignment, where the model is fine-tuned to be helpful, honest and safe. The system prompt is set by the application (like Claude.ai) to define the AI's persona and rules for every conversation. Finally, the user layer is where you interact — your prompts sit at the top, but they're shaped by everything beneath.
  • Context engineering vs. prompt engineering
    Prompt engineering is about crafting a single well-worded request. Context engineering is the broader discipline of managing everything the model can "see" at once — including documents, memory, tool results, and conversation history. As AI workflows grow more complex, context engineering becomes the more powerful skill: you're not just writing a better question, you're curating the full information environment the AI reasons within.
  • Comparing Claude, ChatGPT, Gemini & Agents
    Each platform has the same core LLM capability but differs in ecosystem: Claude excels at long documents and nuanced reasoning with Skills/MCP; ChatGPT has a broad plugin marketplace and persistent memory; Gemini integrates natively with Google Workspace. Agents go further — they don't just answer, they plan and take multi-step actions autonomously, like browsing the web, writing files, or calling APIs in sequence.
  • Knowledge representation in AI systems
    Unlike traditional databases that store facts in tables, LLMs compress knowledge into billions of numerical weights — a form of implicit, probabilistic memory. This means they can reason flexibly across domains but can also "hallucinate" when knowledge is sparse or ambiguous. Understanding this distinction helps you know when to trust the model's built-in knowledge vs. when to feed it verified external data through RAG or MCP.

Day 2

Artifacts, MCP & Skills

Artifacts MCP Skills Promptuary

Topics covered

  • Gen 1–4 artifacts: documents → interactive apps → AI-powered tools → inline
    Artifacts started as a simple sidebar panel for generated documents (Gen 1). They evolved into interactive HTML/JS mini-apps like games and dashboards (Gen 2), then gained the ability to call the Claude API directly — meaning the artifact itself can run AI logic, not just display static content (Gen 3). The latest Gen 4 renders artifacts inline in the conversation, making the boundary between chat and app essentially disappear.
  • MCP as universal AI interface & the Promptuary system
    Model Context Protocol (MCP) is an open standard (launched Nov 2024) that gives AI a universal "hands" — a consistent way to call external tools, read data sources, and take actions regardless of which service is on the other end. Think of it like USB-C for AI integrations. Promptuary is a practical MCP application: a prompt library system organized by PMBOK performance domains, accessible directly from within a Claude conversation so your best prompts are always one step away.
  • Skills: reusable instruction packages (SKILL.md)
    A Skill is a packaged set of custom instructions stored in a SKILL.md file that Claude reads and follows whenever triggered. Unlike a one-off prompt, Skills persist across all sessions and can be shared org-wide. They use progressive loading — only reading the parts of the skill actually needed for a given task, keeping context window usage lean. Anthropic also used this same mechanism internally to build Claude's own file-creation capabilities.
  • Intent matching vs. explicit slash commands
    Skills can be triggered in two ways. Intent matching is the smarter path: Claude reads your request, scans available skill descriptions, and automatically applies the right skill — no trigger word needed. For example, asking "clean up this text" automatically invokes a polish skill. Slash commands give explicit control: typing /risk-register directly invokes that skill. Anthropic recently merged both approaches so you can use either interchangeably.
  • Skill Creator: building custom skills with Claude
    You don't need to write SKILL.md files by hand. The Skill Creator skill is a meta-skill — a skill that helps you build other skills. Describe your workflow to Claude (e.g. "I always want status reports in this format with these sections"), and it will structure, write, and package the skill for you. You can then upload it to your personal or org skill library, toggle it on, and it's live across every future conversation.

Day 3

Advanced Tools & Integration

NotebookLM Cowork Security

Topics covered

  • Video & audio generation with NotebookLM
    Google's NotebookLM can transform your documents, notes, and research into audio overviews — podcast-style conversations between two AI hosts that summarize and discuss your content. This is a powerful way to make dense project documentation accessible and shareable: imagine turning a 50-page feasibility study into a 10-minute audio briefing your team can listen to on the go.
  • Claude Cowork: no-code desktop task automation
    Claude Cowork is a desktop tool that brings agentic AI capabilities to non-developers — think of it as Claude Code without needing to know how to code. It can create and edit files (Word docs, spreadsheets, presentations, PDFs), automate multi-step workflows, and manage tasks across your local system. It's the practical bridge between "AI gave me an answer" and "AI did the work for me."
  • MCP vs. Skills — when to use which
    Skills encode how Claude should think and work: methodology, tone, output format, evaluation criteria. MCP controls what Claude can access: live databases, Slack channels, spreadsheets, external APIs. MCP also acts as a deterministic limiter — it constrains Claude's reach so it only touches what you've explicitly permitted. Use Skills for repeatable logic; use MCP for live data. The most powerful setups combine both: MCP pulls the data, Skills shape what Claude does with it.
  • Extending AI as a hub vs. embedding AI as a feature
    There are two fundamentally different ways to integrate AI into your work. Extending AI (the hub model) puts the AI at the center: you talk to Claude, and Claude talks to all your tools — Slack, Google Drive, your database, your calendar. You get a single interface for everything. Embedding AI (the feature model) puts AI inside existing apps: Google Docs gets a "Help me write" button, Excel gets Copilot. Both have value, but the hub model is more powerful for complex, cross-tool workflows.

MCP vs Skills — the core distinction

Skills — teach Claude how to work

Methodology · tone · format · metrics · logic

  • Packaged prompts & instructions in SKILL.md files
  • Define methodology, tone, output format, evaluation criteria
  • Persist across all chat sessions — no re-explaining ever
  • Triggered automatically via intent matching or slash commands
  • Admins can publish org-wide shared skill libraries
  • Can include executable scripts for technical workflows
Example: a risk-register skill that always outputs probability/impact scores, trigger conditions, and mitigation plans in your org's exact standard format

MCP — define what Claude can reach

Data sources · tools · external systems · boundaries

  • Connects Claude to external tools & live data sources
  • Pull real-time data: databases, spreadsheets, Slack, APIs
  • Acts as a limiter — ensures deterministic, controlled behavior
  • Connectors (remote/cloud) vs. Extensions (local machine)
  • Granular permission management per individual tool action
  • Open standard — any service can build an MCP server
Example: pulling Q3 actuals from a database and reading the latest Slack standup before Claude generates an executive status report
Rule of thumb: Use Skills to encode repeatable logic — how to structure a report, what methodology to follow, tone and format. Use MCP to feed Claude real-world data — what numbers to use, where to look, what actions it's permitted to take. They work best together: MCP fetches the live data, Skills determine how Claude processes, evaluates, and presents it.

Key learnings

🧠

Context is your most valuable resource

The quality of AI output is determined less by the model's intelligence and more by the quality of context you provide. Strategically include what's relevant, exclude what's noise, separate unrelated conversations, and refresh key instructions in long sessions. Context engineering is the new competitive advantage.

🔗

AI as a hub changes everything

The shift from "AI as a chatbot" to "AI as the central hub of your workflow" is profound. Instead of juggling 10 disconnected tools, you interact with one AI that orchestrates all of them via MCP. This is the LLM OS concept: AI becomes the universal coordinator, and your tools become its peripherals.

📦

Skills make AI behave consistently

The biggest friction in daily AI use is re-explaining your preferences, format requirements, and methodology in every new chat. Skills solve this permanently. Once you encode a workflow — a risk assessment process, a charter template, a vendor scoring rubric — it's always on, always consistent, and available to your entire team.

⚡

Artifacts democratize app building

Gen 3 AI-powered artifacts mean that anyone in your org can now build interactive tools — dashboards, assessments, trackers — without writing code. The workflow is: describe what you need → Claude builds it → share the link. This flattens the gap between "having an idea" and "having a working tool" from weeks to minutes.

🔄

MCP vs Skills: complementary not competing

A common confusion: Skills and MCP solve different problems. Skills encode logic (how Claude should think, what format to use, what criteria to apply). MCP controls access (what data Claude can read, what actions it can take, what systems it can touch). The most effective AI setups combine both: MCP brings in live data, Skills shape what Claude does with it.

🔐

With more power comes new security risks

As AI gains the ability to take real-world actions — sending emails, modifying files, querying databases — the attack surface grows. Prompt injection (malicious instructions hidden in content the AI reads) is a real threat. Best practice: treat all external content as untrusted, use minimal permissions, require human approval for irreversible actions, and audit your connected MCP servers regularly.
